Setup Instructions (Shizuku Integrated)

  1. Install the Shizuku app from Google Play Store on your Android device
  2. On your device, enable Developer Options: Settings > About Phone > tap Build Number 7 times
  3. Go to Settings > Developer Options > enable Wireless Debugging
  4. Open Shizuku app, tap "Start via Wireless debugging", then tap "Pairing" and enter the pairing code
  5. Create new Android Studio project with minimum SDK 24
  6. Copy PrivateDNSTile.kt to src/main/java/com/example/privatedns/tile/
  7. Copy DNSProviderActivity.kt to src/main/java/com/example/privatedns/
  8. Copy PrivateDNSUserService.aidl to src/main/aidl/com/example/privatedns/
  9. Copy the layout XML to res/layout/activity_dns_provider.xml
  10. Add activity declaration and service to AndroidManifest.xml
  11. Add Shizuku dependencies to build.gradle
  12. Build and run: ./gradlew installDebug
  13. App will automatically request Shizuku permission on first launch
  14. Open Quick Settings (swipe down twice), long-press to add "Private DNS" tile
  15. First tap: Enables DNS with saved provider (shows subtitle)
  16. Second tap: Opens provider selection activity to choose or switch providers

Troubleshooting

📋 Tile doesn't appear in Quick Settings

Ensure the service is properly declared in AndroidManifest.xml, AIDL is compiled, and the app is installed. Restart your device.

❌ Provider activity doesn't open

Verify DNSProviderActivity is declared in AndroidManifest.xml with android:exported="true". Check Logcat for errors: adb logcat | grep PrivateDNS

❌ Shizuku permission request fails

Verify Shizuku app is installed and running. On device, go to Shizuku app > Settings > Grant permission to your app. Use wireless debugging or USB debugging.

⚡ DNS changes not taking effect

Some devices cache DNS settings. Restart your device, toggle WiFi, or clear DNS cache with: adb shell cmd netd resolver flushnet

🔒 AIDL compilation errors

Ensure buildFeatures { aidl true } is in build.gradle and AIDL file is in correct path: src/main/aidl/com/example/privatedns/

⚡ Provider list doesn't populate

Check that SharedPreferences is correctly storing provider data. Verify in Android Studio: Device File Explorer > data/data/[PACKAGE_NAME]/shared_prefs/PrivateDNSPrefs.xml

📱 Theme looks wrong (white text on white)

Ensure the activity theme is set to @style/Theme.AppCompat.DayNight in AndroidManifest.xml. This auto-switches between light and dark themes.

🚀 Shizuku connection drops after reboot

This is expected—Shizuku must be restarted after device reboot. On device, open Shizuku app, tap "Start via Wireless debugging" again (wireless pairing persists).

How It Works: User Flow

First Tap on Tile (When OFF)

Toggles Private DNS ON with the last selected provider (or default AdGuard). Subtitle shows current provider.

Second Tap on Tile (When ON)

Opens DNSProviderActivity where users can:

  • View list of 6+ DNS providers with use case labels
  • Select a provider with radio button (single choice)
  • Changes apply immediately to system Private DNS settings
  • Selection is saved to SharedPreferences
  • Click "Disable Private DNS" button to turn OFF

Persistent Selection

The app saves which provider was last selected. When user enables DNS again, it uses that provider automatically.

Shizuku Integration

No ADB Required: Users install Shizuku app once, enable wireless debugging, pair device. Your app automatically requests permission—no command-line needed.
